Vertigo

Dynamic blog engine in Go
Install - Deploy to Heroku

Vertigo is a dynamic blog engine written in Go. Its key feature is the JSON API, which makes extending Vertigo easy.

You can create plugins, SPA's and retrieve your blog's data in programmatic form using plain HTTP requests, without knowledge in Go. Fetching all users and their posts would be issued like so:

HTTP GET yourblog.com/api/users

To see the response data, you can use my blog as an example: juusohaavisto.com/api/users

Registering a new account would be as easy as:

HTTP POST yourblog.com/api/user

{
	"name": "Foo",
	"password": "bar",
	"email": "foobar@example.com"
}

The frontend code is powered by Go's template package, which syntax is similar to Handlebars.

My blog's /templates/home.tmpl

{{range .}}
{{if .Published}}
<article>
	<h1><a href="/post/{{.Slug}}">{{.Title}}</a></h1>
	<p>{{unescape .Excerpt}} [...] <a href="/post/{{.Slug}}">Read more »</a></p>
</article>
{{end}}
{{end}}

The template files are in plain HTML and JavaScript (vanilla) only appears on few pages. JavaScript in general is aimed to be stripped down as much as possible to provide better user experience on different devices. Vertigo also ships without any CSS frameworks, so it is easy to start customizing the frontend with the framework of your choice.

For writing blog posts you can either use HTML with bit tricky contenteditable or just bash away using Markdown. Both options will work regardless what device you are using. Backups are stored in your browser's local storage, so changes are saved even if you lose connection or your computer crashes.

Since Vertigo is built in Go, it is also very portable. You can download binaries or build it from the source on Windows, Mac OSX, Linux and FreeBSD. Vertigo can also be built and run on ARM, thanks to the SQLite driver.

Thanks to Go Vertigo is also fast. My blog, running on Heroku's development plans (for free), ranks in the top 1% of websites in speed.
